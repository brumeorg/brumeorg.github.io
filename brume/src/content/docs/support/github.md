Brume is developed in the open on GitHub.

## Project

- **Repository** — [github.com/brumeorg/Brume](https://github.com/brumeorg/Brume)

The repository hosts the source code, the release artifacts, the issue tracker, and the discussion forum.

## Issues

Use **issues** for:

- Bug reports — please include your Brume version (`brume --help`), Postgres version, and the minimal `brume.yml` that reproduces.
- Feature requests — check the [roadmap](/roadmap/) first to see if it's already planned.
- Documentation feedback — typos, unclear sections, missing pages.

→ [Open an issue](https://github.com/brumeorg/Brume/issues/new)

## Discussions

Use **discussions** for:

- Usage questions ("how do I pseudonymize this kind of schema?")
- Compliance questions ("how do I justify this in front of my DPO?")
- Showcase ("here's how we wired Brume in our CI")

→ [Start a discussion](https://github.com/brumeorg/Brume/discussions)

## Releases

Every Brume release is tagged and published on the [Releases page](https://github.com/brumeorg/Brume/releases). The changelog lives there.

## Security

Please **don't open public issues for security vulnerabilities**. The repo's security policy describes how to report them privately.
